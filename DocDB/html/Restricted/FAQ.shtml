Document Database FAQ
These are the frequently asked questions about the Document Database (or at least the ones we feel should be asked frequently).

Why can't I just type in the name of an author on a document?
For technical reasons which are described in BTeV-doc-140, the names of the authors of a document aren't stored in the same place as things like the title of the document. Instead, numbers are stored in the database that link the authors with a particular document. This means each author of a document must be in the master list of authors before you can assign them to have written a document. In the future, this master list will probably be linked with the People Database. Until then, you can add an author yourself.

How do I add a new author? A new topic or subtopic? A new document type.
To add a new author, you can either send e-mail to btev-docdb@fnal.gov or add the person yourself with the form. Please be careful and if you make a mistake, e-mail btev-docdb@fnal.gov rather than trying to fix the mistake.

To have topics or document types added, e-mail btev-docdb@fnal.gov. Be prepared to have your request denied. Having a new document type or major topic added is hard, since having too many of these is confusing. Be prepared to argue why the existing topics or document types don't fit the kind of documents you want to add. Having a new subtopic added is a little easier, but if you want to organize things below the subtopic level, use keywords instead.

Why are there documents with a "Version 0"
The 0th version of a document only exists when a document is reserved, as opposed to being created new. A reserved document usually means one of two things. Either the author intends to create additional versions but wants a reference immediately, or the original files from the document have been lost before the document was migrated into the database.

Why are some document #'s missing?
Two possible reasons. First, it could have been deleted because of an entry error. The information almost certainly exists as another document. Second, the document might exist but you don't have priveledges to view it.

How do I delete a document?
This function is not available to regular users. Send an e-mail to btev-docdb@fnal.gov explaining the situatoin and they will delete the document for you.

What are keywords? How do I use them?
Keywords are really just a text field you can type stuff in, but to make them useful some conventions have been adopted that you should follow:
  • Be consistent. Re-use old keywords. If your subgroup has a defined set of keywords, use them.
  • Keywords are space seperated. Don't use commas. Multi-word keywords should be joined with an underscore or dash. (Again, be consistent).
  • Don't use keywords like "pixel." There are topics for that.

Why do I keep having to type in the password?
First, you should only have to type in the password one extra time for most uses. The reason for this is that people other than those in BTeV use the database too. Currently, this is just the RTES group.

To get technical, because the DocDB can be accessed with more user/password combinations than just BTeV, the Document Database scripts use a different authorization realm. The authorization realm for BTeV only documents matches the realm for the BTeV internal web pages.

However, for documents protected for sub-groups, the realm is different again, so you will need to type an accepted password again.

Using a web browser that saves passwords, like recent versions of IE, or Netscape 6 (which can encrypt the passwords with a master password) takes a lot of the headache out of this process.

Can our sub-group use the DocDB to store documents that aren't accessible to the rest of the collaboration?
Yes. Just ask to have another "group" set up, choose a password, and let the DocDB administrators know which accounts you want to allow access to your data.

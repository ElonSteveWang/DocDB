This document lays our a rough road map of where DocDB should be headed in the next months. As always, this is subject to modification due to our needs and the needs of those contracting for improvements.

  • 6.0 release
    • Add certificate based authentication and with that the concept of individuals that belong to groups.
    • Add CSS (Cascading Style Sheet) framework and a new CSS based layout for ShowDocument.
  • 6.x releases
    • Move over to single ListBy script, deprecate others. (done, 6.0)
    • Document cross-referencing. (done, 6.1)
    • Individual document notification. (done, 6.2)
    • Single session meetings (simplified creation).
    • CSS for more scripts.
  • 7.0 release
    • Truly separate meetings and topics. List meetings by MeetingGroup type.
    • Delete other listing scripts.
    • Split into first/last name, store CN seperately in EmailUsers
    • Allow changing of document types
    • Fix validation of JS, move JS to files
  • Possible in future releases
    • Use CGI::Pretty
    • Allow cross-referencing between instances of DocDB
    • Individual logins with Basic HTTP (rather than groups)
    • From Erik Gottschalk:
      I have a suggestion for the Doc DB. How about a meeting-minute template generator. This would generate an e-mail message that would serve as a template for meeting minutes. It could include a link to the meeting in the Doc Db, links to the individual talks, and a field that would let someone add comments on any discussion that took place during the meeting (or during a particular presentation). At the conclusion of the meeting it could get mailed (automatically) to the person responsible for meeting minutes (or the person who set up the meeting). They could then edit the e-mail (adding the minutes), and post it back into the document database as a document. Anybody who has signed up for receiving new postings by e-mail will then receive e-mail telling them about the new document, and it will have all the links to the talks in case they want to see what they missed.

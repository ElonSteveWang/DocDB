CPAN note

All perl modules needed by DocDB can be downloaded from CPAN.

In the "scripts" directory of the DocDB distribution is a program called CheckModules. This will determine if all the perl modules needed to run DocDB are present on your system and tell you which ones are still needed. You should run this script before attempting to install DocDB. You should also run this script when performing a major upgrade of the DocDB software. The upgrade instructions will tell you when to do this.

To install a CPAN perl module manually:

  • setenv LANG en_US (Some versions of perl on Redhat will fail if you don't do this)
  • setenv LANGVAR en_US (some reports say this is needed in some instances)
  • perl -MCPAN -e shell
    cpan> install module

If necessary, use the old method to download and build a CPAN module.

  • download the module (module.tar.gz)
  • tar -xzf module.tar.gz
  • Before proceeding, make sure your default perl is the one you will be using for DocDB.
    Type which perl and inspect the result.
    Make sure that the result of which perl is the same perl that DocDB will be using.
    This is especially important if you have the ups/upd perl installed.
  • cd module
  • perl Makefile.PL
  • make
  • make test
  • make install
    This is the step that actually installs the perl module in your system files.
